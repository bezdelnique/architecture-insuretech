# Типы данных
- Данные по продуктам и тарифам [core-app, ins-comp-settlement] -> [ins-product-aggregator]
- Оформленные страховки за день [core-app] <- [ins-comp-settlement]

# Риски 
При взаимодействии [core-app, ins-comp-settlement] -> [ ins-product-aggregator] 
- Ответ приходит по скорости самого медленного ответа
- В случае ошибки на стороне мы не получаем часть данных или вообще ничего не получаем в зависимости о того как обрабатываются ошибки 
- В БД core-app который формирует данные для клиентов данные всегда не актуальны
- Сервис будет ухудшаться при подключении новых источников данных

При взаимодействии с [core-app] <- [ins-comp-settlement]
- Основная проблема это задержка с актуальностью данных которую можно избежать

# Решение
## Для работы с данными: Данные по продуктам и тарифам
На уровне ins-product-aggregator
- Опрашивать каждого поставщика так часто как позволит его API с поправкой на частоту обновлений данных на его стороне
- Узнать если есть возможность получать уведомление со стороны поставщика об изменениях и реализовать обратную интеграцию
- При получении данных любым способом создавать событие: Изменение тарифа, Новый продукт и т.д.

На уровне core-app, ins-comp-settlement
- Сделать обработку события: Изменение тарифа, Новый продукт и т.д.

## Для работы с данными: Оформленные страховки за день
- При получении данных любым способом создавать событие изменения данных
- Сделать обработку получения: Новая страховка

На уровне ins-comp-settlement
- Сделать обработку события: Новая страховка
